protoc-gen-flowtypes
====================

Generate flowtype type definitions for proto3 messages and enums.

Contributions welcome.

```sh
$ cat simple.proto
```
```proto
syntax = "proto3";

...

message SearchRequest {
  string query = 1;
  int32 page_number = 2;
  int32 result_per_page = 3;
  enum Corpus {
    UNIVERSAL = 0;
    WEB = 1;
    IMAGES = 2;
    LOCAL = 3;
    NEWS = 4;
    PRODUCTS = 5;
    VIDEO = 6;
  }
  Corpus corpus = 4;
  int64 example_required = 5 [(validator.field) = {msg_exists : true}];
  google.protobuf.Timestamp sent_at = 6;
}

message SearchResponse {
  repeated string results = 1;
  int32 num_results = 2;
  SearchRequest original_request = 3;
}
```

# example use:
```sh
$ protoc -I. -I${GOPATH}/src --flowtypes_out=. simple.proto
$ cat simple_types.js
```

# [simple_types.js](simple_types.js):
```js
/* @flow */
/* eslint-disable */
// Code generated by protoc-gen-flowtypes DO NOT EDIT.
// InputID: 351d5461ac8b58d4235f96c4fdec32a53a0e4f43

export type SearchRequestCorpus = "WEB" | "IMAGES" | "LOCAL" | "NEWS" | "PRODUCTS" | "VIDEO";

export type SearchRequest = {
  query?: string,
  page_number?: number,
  result_per_page?: number,
  corpus?: SearchRequestCorpus,
  example_required: number,
  sent_at?: string
};

export type SearchResponse = {
  results?: Array<string>,
  num_results?: number,
  original_request?: SearchRequest
};

```
